# Rename Obsidian → Aniani Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Rename the project from `obsidian` to `aniani` across code, docs, CI, the working directory, and the GitHub repo, including runtime default paths.

**Architecture:** This is a mechanical, case-preserving find-replace over a scoped file set, gated by the existing `cargo build` + `cargo test` suite (the rename's safety net). Irreversible/outward-facing steps (GitHub repo rename, directory move) are sequenced last behind confirmation. Historical records under `docs/superpowers/` and build artifacts are deliberately left untouched.

**Tech Stack:** Rust (cargo), `perl -pi` for case-aware replacement, `git`, `gh` CLI.

**Spec:** `docs/superpowers/specs/2026-06-06-rename-obsidian-to-aniani-design.md`

**Case mapping (applied everywhere):** `OBSIDIAN`→`ANIANI`, `Obsidian`→`Aniani`, `obsidian`→`aniani`

**Deliberately NOT touched:** `target/`, `dist/`, `libnull.rlib`, `Cargo.lock` (regenerated by build), and `docs/superpowers/**` (dated historical specs/plans, including this file and the prior `2026-03-28-adversarial-review-fixes.md`).
