# Rename: Obsidian → Aniani

**Date:** 2026-06-06
**Status:** Approved (name + scope) — pending spec review

## Decision

Rename the project from **Obsidian** to **Aniani**.

`aniani` (Hawaiian: *mirror; clear, transparent glass*) succeeds the original
Obsidian metaphor — "dark glass that reflects the system back to you" — flipped
from dark to clear. It is the heir to the same idea, with none of the baggage.

## Why rename

`Obsidian` collides hard with [Obsidian.md](https://obsidian.md), the wildly
popular note-taking app. The name is overloaded for search, recognition, and any
future crates.io / GitHub / domain identity. DESIGN.md already flagged the name
as a placeholder.

## Why Aniani (availability findings, checked 2026-06-06)

- **crates.io** — `aniani` is available; zero close crate names.
- **No soundalike trap** — unlike the runner-up `Akaka` (a near-homophone of the
  JVM toolkit **Akka**, which now also markets itself around agentic AI),
  `aniani` has no collision in the dev/infra space. Nearest tech terms
  (App Annie, Aaniie, telephony "ANI") are clearly distinct.
- **No real project collision** — GitHub `aniani*` repos are all zero-star noise.
- **ASCII-clean** — lowercase, no diacritics; drops straight into `Cargo.toml`.
- **Note**: the bare `github.com/aniani` username is taken (inactive). Not needed —
  the repo lives under the existing `tritonrc` org as `tritonrc/aniani`.
- Benign associations: Akaka Falls (clarity), authentically Hawaiian.

## Scope (approved)

**Code + docs + directory + git**, including renaming runtime default paths.

## Behavioral consequences

- A renamed binary will no longer auto-load snapshots from `.obsidian/obsidian.snap`;
  it looks under `.aniani/aniani.snap`. Acceptable — instances are ephemeral.
- CLI program name in `--help` becomes `aniani`.
- Released artifact filenames change (next release only).

## Risks & mitigations

- **Crate rename breaks all `use` paths** → handled by the 20-file sweep; caught by `cargo build`.
- **Directory rename while it is the shell CWD** → do this step last; the session
  may need to reopen at the new path afterward.
- **GitHub repo rename** is outward-facing → confirm before running; GitHub keeps
  redirects so existing clones/links keep working.
- **Stray case variants** → final `grep -ri obsidian` sweep (excluding `target/`,
  `dist/`, `libnull.rlib`) must return clean.

## Verification

- `cargo fmt --check` and `cargo clippy` pass
- `cargo build` succeeds (regenerates `Cargo.lock` with `aniani`)
- `cargo test` passes (proves crate-path rename is complete)
- `./target/debug/aniani --help` shows `aniani`
- `grep -ri obsidian . --exclude-dir=target` returns nothing (except this spec's history)
